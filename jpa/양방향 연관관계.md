# 양방향 연관관계

엔티티의 관계가 서로를 참조하게되면 양방향 연관관계라고 한다.

객체에서는 Member는 Team을 가지고있고 Team은 일대다 관계여서 여러Member와 관계를 맺을 수있으므로 List 컬렉션을 사용한다.  
테이블에서는 외래키 하나로 Join을 통해 Member와 Team을 조회가능하기때문에 단방향 연관관계와 동일하다.

## 객체 관계 매핑

```java
@Entity
public class Member {
  @Id @GeneratedValue
  private Long id;

  @Column(name = "USERNAME")
  private String name;

  @ManyToOne
  @JoinColumn(name = "TEAM_ID")
  private Team team;
  ...
}
@Entity
public class Team {
  @Id @Column(name = "TEAM_ID")
  private String id;

  private String name;

  @OneToMany(mappedBy="team")
  private List<Member> members = new ArrayList<>();
}
```

Team에 List로 컬렉션이 추가되었다.  
@OneToMany: 일대다 관계를 의미하며 Team에게 Member는 다대일이다. 그리고 mappedBy속성으로 연관관계의 주인이 team임을 명시한다.

# 연관관계의 주인

- 양방향 매핑의 규칙
  - 두 객체의 연관관계중 하나를 연관관계의 주인으로 정한다.
  - 연관관계의 주인만이 외래키를 관리할 수 있다.(등록, 수정, 삭제)
  - 주인이 아닌쪽에서는 조회만 가능하다.
- 연관관계의 주인은 외래키가 있는곳으로 정해야한다.
  - 다대일, 일대다 관계에서는 항상 다 쪽이 외래키를 가진다.

# 양방향 연관관계 매핑시 주의할점

- 연관관계의 주인에 값을 입력해야한다.
